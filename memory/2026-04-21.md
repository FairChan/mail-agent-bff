## Registration SMTP DNS Fallback

- Investigated a WebUI registration failure that surfaced as `Failed to send verification email. Please check email settings and try again.`
- Root cause in BFF logs: transient DNS failure resolving `smtp.gmail.com` (`getaddrinfo ENOTFOUND smtp.gmail.com`), not a bad Gmail app password or broken registration logic.
- Added host-scoped SMTP transporter caching and Gmail fallback SMTP host support in `apps/bff/src/email.ts`; when `smtp.gmail.com` fails after retries, BFF tries `gmail-smtp-msa.l.google.com`.
- Hardened the registration live smoke so Gmail plus aliases include timestamp plus random bytes and session transitions are asserted, not merely printed.
