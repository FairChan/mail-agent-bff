# 2026-04-17

## Microsoft Outlook Direct Auth

- Fixed the Outlook direct-auth callback bug where Microsoft redirected to `localhost:8787` and the BFF global `/api/*` auth hook returned raw JSON `{"ok":false,"error":"Unauthorized"}` before the callback route could validate OAuth `state`.
- Root cause: the OAuth popup starts from `127.0.0.1:5173`, but Microsoft callback uses `localhost:8787`; callback cannot rely on the host-only browser cookie and must rely on one-time OAuth `state` tied to the original session.
- Code changes:
  - Allowed `/api/mail/connections/outlook/direct/start` and `/api/mail/connections/outlook/direct/callback` through the global auth hook.
  - Kept start route protected with route-level session validation and rate limiting.
  - Added `touchAuthSessionForRequest()` so direct auth reuses Redis hydration/revocation behavior.
  - Added pre-token-exchange and pre-token-persist session checks for Microsoft callback.
- Validation passed: BFF check/build, targeted semantic check, health checks, callback HTML behavior, authenticated start redirect, and `npm run harness:smoke`.

## Email Verification Registration

- Implemented the active registration flow in `apps/bff/src/server.ts`: register now creates an in-memory pending registration, sends/logs a 6-digit verification code, and only creates the real user after `/api/auth/verify` succeeds.
- Added `/api/auth/resend` with cooldown and rate limiting, and bounded/expired pending-registration cleanup.
- Wired WebUI registration into a verify step using `AuthContext` and `ContextAuthScreen`.
- Configured ignored local Gmail SMTP defaults for `mery.secretary@gmail.com` using `smtp.gmail.com:465`, but kept `SMTP_ENABLED=false` until the user provides a Gmail App Password or OAuth refresh token.
- Sub-agent audit found no Critical/High issues. Deferred medium: pending registrations are memory-only and need Redis/DB persistence before production/multi-instance use.
- Validation passed: BFF/WebUI check/build, targeted semantic, runtime register -> log code -> verify -> authenticated session, smoke 10/10, and `check:standard`.
