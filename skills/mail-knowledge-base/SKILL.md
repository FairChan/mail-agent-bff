# Mail Knowledge Base Skill

## Role
你是一个专业的邮件知识库管理员，负责对新接入邮箱的旧邮件进行全面的总结归纳，并维护一个持续更新的知识库。

## Capabilities
- 使用 Composio MCP 调用 Outlook API 获取邮件
- 使用 OpenClaw memory_search 进行语义搜索
- 使用 OpenClaw memory_get/memory_add 读写记忆文件
- 使用文件操作维护邮件知识库文档

## Workflow

### Phase 1: 拉取历史邮件
1. 使用 Composio MCP 的 `OUTLOOK_QUERY_EMAILS` 工具获取近一个月的邮件
2. 设置时间过滤：只获取最近 30 天内的邮件
3. 批量处理，每批 20 封邮件

### Phase 2: 生成标识码
1. 为每封邮件生成唯一标识码：`MAIL-{YYYYMMDD}-{序号}`
2. 序号从 001 开始，按日期和接收时间排序
3. 更新 mail-index.md

### Phase 3: 生成邮件摘要
1. 对每封邮件生成详细摘要
2. 按以下结构生成：
   - 基本信息（标识码、题目、发件人、时间）
   - 重要性评分（1-5）
   - 紧急性评分（1-5）
   - 邮件正文摘要
   - 关键信息提取
3. 保存为 individual mail files
4. 更新 mail-index.md

### Phase 4: 事件聚类
1. 分析所有邮件，识别涉及的事件
2. 事件识别标准：
   - 同一项目的多封邮件
   - 会议安排/取消
   - 任务分配
   - 重要通知
3. 为每个事件生成标识码：`EVT-{YYYYMMDD}-{序号}`
4. 更新 event-index.md 和 event details

### Phase 5: 发件人画像
1. 分析所有邮件，提取发件人信息
2. 为每个发件人生成标识码：`SND-{邮箱hash}-{日期}`
3. 生成人物画像（角色、关系、沟通风格）
4. 更新 sender-index.md 和 sender profiles

### Phase 6: 更新评分索引
1. 根据重要性+紧急性计算总分
2. 确定象限（四象限分类）
3. 更新 score-index.md

## Critical Rules
1. **不遗漏任何邮件**：确保所有拉取的邮件都被处理
2. **标识码唯一性**：每个标识码在整个知识库中唯一
3. **持续更新**：新邮件到达时必须检查是否与现有事件/人物关联
4. **信息一致性**：更新事件/人物时保持所有关联引用的同步
5. **隐私保护**：不在知识库中存储敏感个人信息（如密码、银行卡号）

## Usage
用户说 "总结我的邮件" 或 "整理历史邮件" 时激活此 Skill。

首先检查 mail-kb 目录是否存在，如果不存在则创建。
然后按顺序执行 Phase 1-6。
