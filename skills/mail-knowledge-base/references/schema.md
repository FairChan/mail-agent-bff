# Mail Knowledge Base Schema

## 标识码格式

### 邮件标识码
- 格式: `MAIL-{YYYYMMDD}-{序号}`
- 示例: `MAIL-20260411-001`
- 生成规则:
  1. YYYY = 4位年份
  2. MM = 2位月份
  3. DD = 2位日期
  4. 序号 = 3位数字，从001开始，按日期排序

### 事件标识码
- 格式: `EVT-{YYYYMMDD}-{序号}`
- 示例: `EVT-20260411-001`
- 生成规则: 同邮件标识码

### 发件人标识码
- 格式: `SND-{邮箱hash}-{YYYYMMDD}`
- 示例: `SND-a3f2b1c4-20260411`
- 生成规则:
  1. 邮箱hash = 邮箱地址的MD5前8位
  2. 日期 = 首次识别日期

## 评分标准

### 重要性评分 (1-5)
- 5: 涉及重大决策、财务、法律事务
- 4: 涉及工作任务、项目重要节点
- 3: 常规工作交流
- 2: 一般性通知
- 1: 垃圾邮件、广告

### 紧急性评分 (1-5)
- 5: 需要立即行动的紧急事项
- 4: 24小时内需要响应的
- 3: 7天内需要响应的
- 2: 有截止日期但时间充裕
- 1: 无截止日期或无限期

### 象限分类
| 重要性 | 紧急性 | 象限 |
|--------|--------|------|
| >=4 | >=4 | 紧急且重要 (Do First) |
| >=4 | <4 | 重要不紧急 (Schedule) |
| <4 | >=4 | 紧急不重要 (Delegate) |
| <4 | <4 | 不紧急不重要 (Eliminate) |

## 文件命名规范
- 所有文件名小写，用连字符分隔
- 邮件详情: mail-{标识码}.md
- 事件详情: event-{事件ID}.md
- 发件人画像: sender-{人物ID}.md
